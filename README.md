# ボイスチェンジャーを作ろう

## 背景と課題

既存のボイスチェンジャーは自分の声に対して個人の特徴情報を残さないようにノイズ処理を行う。
具体的に、周波数を再生速度を落とすことで下げ、音の高さを低くする方法などがある。

また、一般的にボイスチェンジャーはその特性から声バレを防ぐために使用される。
低い男性の声に変えるボイスチェンジャーではいかなる理由で使用したとしても、
事件の犯人のように思われてしまう。これは、会話相手に不安感を持たせる原因になっている。

## 提案手法

本実験では、会話相手を安心させるボイスチェンジャーを提案する。
単に声を低くする訳ではないボイスチェンジャーは多く存在する。
今回は、現代の若者にもっとも親しまれている「ずんだもん」の音声に変化させる。

## 実装内容

音声認識によるテキスト抽出と元の声のテンションの抽出を行い、VOICEVOXエンジンに流し込む。

### 使用している主な技術・ライブラリ

- マイク入力・音声再生: sounddevice, soundfile, subprocess (Mac標準の afplay)
- 文字起こし・時間抽出: openai-whisper
- ピッチ（音程）抽出: librosa (pyinアルゴリズム)
- テキスト解析（フリガナ変換）: pykakasi
- 音声合成: VOICEVOX (ローカルAPI)

### 処理のパイプライン

本システムは、以下の5つのステップで処理している。

1. マイク入力フェーズ
sounddevice を使用して、指定したデバイス（MacBook Proのマイク等）から数秒間ユーザーの音声を録音し、一時的なWAVファイル（your_voice.wav）として保存します。

2. データ抽出フェーズ（文字と音程の取得）
Whisper: 録音データを解析し、発話された「テキスト」と、単語ごとの「開始・終了のタイムスタンプ」を取得します。
librosa: 録音データの音声波形から、時間の経過に伴う基本周波数（F0 = 声のピッチ推移）をHz単位で抽出します。

3. アライメントフェーズ（時間と発音の同期）
漢字混じりのテキストをそのまま使うと発音の単位（モーラ）とズレるため、pykakasi を用いてテキストをすべて「ひらがな」に変換します。
「ぁ・ぃ・ゃ」などの小書き文字を計算から除外することで、VOICEVOXが認識する正確な「モーラ数（音符の数）」を算出します。
Whisperから得た単語の時間をモーラ数で均等に分割し、それぞれの文字が発音されたタイミングの「ユーザーのピッチ（Hz）」を抽出・マッピングします。

4. 整形・マッピングフェーズ（ハイブリッド方式）
ユーザーの声の上がり下がりをそのまま適用するのではなく、「イントネーション（メロディの形）はずんだもんのデフォルトを使い、アクセント（一番高い音のテンション）だけをユーザーの声と同期させる」というハイブリッド計算を行います。
フレーズごとにユーザーの声の最大ピッチを取得し、全体の平均ピッチからの差分（テンションの高さ）を計算します。
その差分をVOICEVOXのデフォルトピッチの推移にオフセットとして加算することで、「ずんだもんの自然な発音」と「ユーザーの感情の起伏」を両立させます。
※低音すぎると発生する「掠れ声（ボーカルフライ）」を防ぐため、ピッチの最低値を 4.5 で食い止める安全装置（リミッター）を搭載しています。

5. 音声合成・出力フェーズ
ピッチ情報が上書きされた専用のJSONデータ（AudioQuery）をVOICEVOXのAPI (/synthesis) に送信します。
返ってきたずんだもんの音声データを zundamon_output.wav として保存し、Mac標準の afplay コマンドで自動的にスピーカーから再生します。

### この実装の優れたポイント

- ハイブリッド・ピッチ制御: 機械的な不自然さを排除し、プロ品質の自然なイントネーションを保ちつつ自分の感情を乗せることができる。
- 堅牢なモーラ計算: pykakasi を挟むことで、長い文章や漢字の読み違いによる「ピッチのズレ・配列エラー」を完全に防止している。
- ノイズ耐性: 80Hz以下の音をプログラム側で「ノイズ・息継ぎ」として切り捨てることで、安定した変換品質を維持している。
